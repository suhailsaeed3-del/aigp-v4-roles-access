1. System overview

The portal is a Next.js 14 application (App Router, TypeScript, RTL Arabic UI) backed by PostgreSQL 16 through Prisma ORM. Users sign in with UAE PASS (OIDC); a server-side sessions table backs the signed-in session. An optional internal, OpenAI-compatible model powers the AI review step. One codebase serves all five roles — مشرف النظام (admin), ممثل الجهة, منسق المسار في الجهة, رئيس المسار, and اللجنة الوطنية; every screen derives from a single role-scoped view-model, so what a user can see is decided in one place. The admin gets a dedicated console for managing users & roles and assigning the stream heads and national committee.

Schema source of truth: prisma/schema.prisma + migrations 0001–0008 (verified on a fresh PostgreSQL 16 — 24 tables). The seed loads reference data + roles + starter accounts; items, launch plans and all transaction tables start empty (the demo portfolio only seeds when NEXT_PUBLIC_DEMO_DATA=1).

8. Access provisioning & the admin console

Accounts live in users (FK users.role → roles.key). Two actors provision people, and the split is enforced by role: the admin owns platform-level identities (stream heads and the national committee), while each entity rep owns the coordinators inside its own entity.

Fig. 7 — Who provisions whom (admin console vs. entity team setup)
